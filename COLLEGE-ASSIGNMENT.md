Pracujemy w zespołach 2 osobowych nad zaprojektowaniem, opracowaniem i wdrożeniem systemu składającego się z serwera z REST API, bazy danych oraz aplikacji webowej lub mobilnej korzystającej z API. Studenci wybierają samodzielnie grupy w ramach projektu. Standardowo praca w grupach 2 osobowych. W grupach z nieparzystą ilością uczestników dopuszczalne grupy 3 osobowe. Jednak zgłoszone zadanie projektowe musi być bardziej rozbudowane - do uzgodnienia z prowadzącym.

Celem projektu jest zrozumienie architektury systemów, protokołów komunikacyjnych oraz zarządzania bazami danych w rzeczywistych zastosowaniach.

Cele według karty przedmiotu

Nauka projektowania i implementacji REST API - dla chętnych FAST API
Zdobycie doświadczenia w pracy z bazami danych do przechowywania i pobierania danych.
Zrozumienie komunikacji między aplikacjami webowymi/mobilnymi a serwerami.
Implementacja uwierzytelniania.
Implementacja lokalnej bazy danych w aplikacjach WEB lub urządzeniach mobilnych.
Rozwój umiejętności pracy zespołowej i zarządzania projektem.
Wymagania Projektowe

Backend:

Implementacja REST API przy użyciu wybranego dowolnego frameworka.
Użycie bazy danych - SQL lub NoSQL.
Zapewnienie operacji CRUD dla podstawowych encji.
Implementacja uwierzytelniania (JWT, OAuth lub uwierzytelnianie oparte na kluczach API).
Implementacja obsługi błędów.
Frontend:

Opracowanie aplikacji Webowej lub mobilnej (dla grupy IAiSM preferowana aplikacja mobilna w Kotlin (Android), Swift (iOS) lub w hybrydzie (Flutter, React Native, Ionic, Xamarin, PhoneGap, Kotlin Multiplatform Mobile itp.) dla grup IAiSC możliwość wyboru pomiędzy aplikacjami WEB lub mobilnej).
Pobieranie i wyświetlanie danych z backendu.
Implementacja uwierzytelniania użytkowników i zarządzania sesją.
Implementacja lokalnego przechowywania danych (np. SQLite). W aplikacjach WEB dowolna baza danych
Zbudowanie prostego interfejsu użytkownika (UI/UX). W przypadku aplikacji WEB responsywne
Ogólna charakterystyka projektu:

Prosty panel logowania z zachowaniem pełnego bezpieczeństwa danych

Dowolna funkcjonalność w zależności od tematu projektu. Tematy będą zgłaszane na pierwszych zajęciach laboratoryjnych. Oceniana będzie też kreatywność studenta.

Połączenie z dowolną bazą danych. Akceptowalne połączenie z firebase, SQLittle, MySQL lub PostgreSQL

W zależności od tematu projektu:

Dodawanie rekordów do bazy danych
Zmiana rekordów w bazie danych
Wyszukiwanie rekordów - kilka opcji wyszukiwania
Pobieranie danych z kilku tabel (w zależności od projektu)
Usuwanie rekordów z bazy danych
Korzystanie z publicznego API np. prognoza pogody, mapy, kanały informacyjne, korzystające z interfejsów API, mogą też być witryny i aplikacje oferujące rezerwację hoteli, lotów czy wynajem samochodów korzystają z interfejsów API

Stwórz własne API (do uzgodnienia przy zgłaszaniu tematów projektów)
